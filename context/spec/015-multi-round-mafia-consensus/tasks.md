# Tasks: Multi-Round Mafia Consensus by Pointing (Spec 015)

Vertical slices for [spec 015](./functional-spec.md) per its
[technical-considerations](./technical-considerations.md) — Phase 5 / **Richer
Night Resolution**. Replace the single-round Night pointing with a **bounded
multi-round consensus loop**: living Mafiosos point in a fresh random order each
round, each pointer sees the picks already made, **unanimous agreement** ends the
Night early, else fall back to **majority of the final round** (random tie-break),
capped at **three rounds**.

Each slice leaves the game runnable: **Slice 1** ships the loop + resolution (the
mechanic works end-to-end — the game is fully playable even before the AI is good
at converging); **Slice 2** gives the AI the running picks so it can actually
converge; **Slice 3** shows the human the convergence in the point modal. Offline
verification is `uv run pytest -q` — the `safe_llm` net + the `fake_large`
pointing fixtures + monkeypatched `_shuffle_order` / `random.choice` keep the suite
model-free (architecture §6 — Determinism Posture & Testing Conventions). The
**central invariant** held throughout: **each pick is its own committed
super-step**, so the human `interrupt()` is replay-safe (no prior AI pick
recomputed on resume) and within-round in-turn visibility is free — the reason the
topology is per-*pointer*, not per-*round*. Agents: `langgraph-agentic`
(topology, state, resolution, prompt), `textual-tui` (the point modal), `testing`
(suites).

## Notes for the implementer

- **The per-pointer topology is the spine.** Each pick committed as its own super-step is what makes the human `interrupt()` replay-safe (a resume re-reads committed state and recomputes no prior AI pick) and within-round in-turn visibility free. A per-*round* node with a mid-loop interrupt would recompute the AI picks made before the human's turn on every resume, drifting from what the human was shown — do not collapse the loop back into one node.
- **Reuse `_shuffle_order`, don't duplicate it** — one shuffle surface means one monkeypatch point for tests (and one place the determinism posture applies). Importing it from `nodes/day.py` or lifting it into a shared helper are both fine.
- **No win-math change** — `check_win_night` is untouched; this only changes *which* victim the Night produces, never the win conditions. The first-Night Mafia intros (`first_night_mafia_intros`) and diaries (`night_close`) are unchanged.
- **No new dependency, no `/awos:hire`** — `langgraph-agentic`, `textual-tui`, and `testing` cover every task; verification is offline `uv run pytest -q` (no browser/MCP needed for a TUI).
