# Tasks: Long-Term Cross-Game Memory & Career Stats

- **Functional Specification:** [`./functional-spec.md`](./functional-spec.md)
- **Technical Considerations:** [`./technical-considerations.md`](./technical-considerations.md)
- **Architecture decision:** [ADR 008](../../adr/008-self-managed-memory-pipeline.md) (*Long-Term Memory via the Self-Managed Pipeline*) — **supersedes ADR 007**. Remote-mode career stats flow through the self-managed strategy's intended pipeline (per-action events → S3/SNS → consumer Lambda → consolidated long-term record). The original "Layer 1 / Layer 2" two-tier framing is no longer in force.

**Verification posture:** all slices are verified with `uv run pytest` — unit tests for pure functions/stores plus Textual `App.run_test()` for UI. This is a TUI + all-mocked pytest project, so there is no browser MCP; real boto3/Bedrock must never be reached from the suite. Terraform is checked with `terraform validate` + `plan` via the `./tf` container wrapper — never `apply` (live apply is run by the user). Slices 1–5 are local-mode and keep the app runnable with no AWS; slices 6–7 delivered the original remote backend (now superseded — see Slice 6's marker); slice 8 reworks the remote backend to the self-managed pipeline per ADR 008.
