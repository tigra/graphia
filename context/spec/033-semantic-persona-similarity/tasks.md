# Tasks: Semantic (Meaning-Based) Persona Similarity (Spec 033)

Three vertical slices: **Slice 1** records *and* displays a semantic (Bedrock-embedding) persona similarity, mocked in the offline suite; **Slice 2** backfills transcript-preserved past runs (real embeddings); **Slice 3** is the out-of-suite effort-not-results validation. The meaning-based counterpart to the lexical measures (specs 031/032); additive — **no `METRICS_VERSION` bump, no ablation flag** (measurement-only).

> **Depends on spec 032** — reuses its **value-type metric shape**, **value-type viewer render**, and **transcript backfill harness**. Implement **after** 032 lands.

> **Architecture-decision flag:** introduces a **cloud embedding model on every measured run** (a deliberate architecture §6 exception — metrics were pure/lexical/local). Reproducible (embeddings are deterministic) but model-dependent. Recommend recording a **`/buddah:adr`** for the model choice + the cross-provider dependency before or alongside implementation.
