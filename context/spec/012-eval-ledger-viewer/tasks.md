# Tasks: Eval Ledger Viewer (Spec 012)

Vertical slices for [spec 012](./functional-spec.md) per its
[technical-considerations](./technical-considerations.md). The deliverable is a
standalone Textual app (`make view-ledger`) that reads `evals/blunder-ledger.yaml`
and shows it as a scrollable, searchable table with a full-record drill-down —
the reader spec 011 deferred. Two layers: a **pure, Textual-free data layer**
(`src/graphia/eval_ledger.py`) and a **thin Textual viewer**
(`src/graphia/ui/ledger_viewer.py`).

Each slice leaves the viewer runnable with more capability (vertical slicing:
MVP table → search → drill-down). Offline verification is `uv run pytest -q` —
pure unit tests over the data layer + Textual `App.run_test()` (Pilot) tests
over the app (the repo's real TUI-test tool, mirroring `tests/test_app_boot.py`);
no LLM, no network, no AWS (the viewer never imports `load_config`). Agents:
`python-backend` (pure data layer, dependency, docs), `textual-tui` (the Textual
app), `testing` (suites). The **headline risk is heterogeneous records** — the
committed ledger already mixes pre-provenance (`run.games`, no `code`, no CI)
with full records, absorbed by the defensive flattener (tech-spec §2.1, §3).
