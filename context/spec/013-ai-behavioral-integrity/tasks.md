# Tasks: AI Behavioral Integrity & Outcome Tracking (Spec 013)

Vertical slices for [spec 013](./functional-spec.md) per its
[technical-considerations](./technical-considerations.md). This is the first
**gameplay-changing** increment since the Day-phase integrity trio — it edits
the real game prompts — bundled (per the user's choice) with the measurement
that proves the fix worked.

**The slice order IS the measure-first discipline** (functional-spec §2.6): the
new tracking lands and a **before-fix baseline is committed on unchanged
behaviour** *before* any prompt change, so the fixes are demonstrated against a
real before-picture — not asserted. Slices: **1** new tracking → **2** capture
the baseline (live) → **3** the prompt fixes → **4** re-measure + compare (live).

Offline verification is `uv run pytest -q` (pure scorers + `render_record` +
`eval_ledger` flatten + Textual `App.run_test()` Pilot — no LLM/network/AWS, the
harness/viewer never reach a real model). The behaviour fixes are
**non-deterministic prompt nudges** verified by the live `make blunder-eval`
before/after (Slices 2 & 4), not pytest. Agents: `python-backend` (harness,
ledger, viewer, docs), `langgraph-agentic` (prompts + nodes), `testing` (suites).

**Two load-bearing constraints carried from the tech spec:**
- **Explicit-zero (vote_activity) is the inverse of absent-omission (metrics):** `vote_activity.by_side` always emits both side keys with integer counts (zero included) — Nova's silent Day must read as a committed `0/0`, never an omitted block.
- **Knowledge boundary:** the role grounding tells a Mafioso its teammates but tells a Law-abiding Citizen **nothing** about any other player's allegiance (see [memory: role knowledge boundary] / functional-spec §2.5). No law-abiding teammate list, ever.
