# Tasks: AI Character Personas (Spec 016)

Vertical slices for [spec 016](./functional-spec.md) per its
[technical-considerations](./technical-considerations.md) — Phase 6 / **AI
Personas & Per-Game Memory**. Give each AI player a distinct persona
(personality, backstory, manner) generated fresh at game start; a **Mafioso**
gets a true self **plus** a public **legend** cover; personas are hidden during
play, shape each AI's voice/temperament, and are revealed at game end.

Each slice leaves the game runnable: **Slice 1** generates + persists personas
and reveals them at game end (the full pipeline proven through the visible
payoff — the game plays normally, personas simply aren't *spoken* yet); **Slice 2**
makes personas shape the Day conversation (in-character voice + the Mafioso
cover); **Slice 3** (optional) adds light persona flavor to Night pointing.
Offline verification is `uv run pytest -q` — the persona LLM call is mocked via
the extended large-model fake, and `safe_llm` is extended to patch the new
`get_large` call site in `setup.py`. The **central invariant** held throughout:
a persona is an **expressive layer only** — it never changes the rules, turn
structure, win conditions, or vote/pointing mechanics; and a Mafioso's `true_self`
lives **only in that Mafioso's own prompt** (the knowledge-boundary invariant,
spec 013). Personas are **transient in-game state** (on `PlayerState`), not
persisted across sessions — no AgentCore Memory. Agents: `langgraph-agentic`
(state, generation node, prompt injection, reveal), `testing` (suites).

## Notes for the implementer

- **A persona is an expressive layer only.** It colours voice and temperament (how readily a character suspects/accuses, how much it says) but **never** changes the rules, the turn order, the win conditions, or any required action — a "quiet" character is quiet in tone, it never skips a turn, vote, or Night pick. No game-flow / graph-topology / win-math change.
- **The Mafioso's `true_self` is owner-private.** It is injected only into that Mafioso's own prompt (with a non-disclosure instruction) and only surfaced publicly at the end-of-game reveal — never broadcast or threaded into another player's prompt. Keeping the legend free of any allegiance tell is **best-effort** (non-deterministic LLM, per CR 005); tests assert the *wiring*, not the model's discretion.
- **`safe_llm` must grow `graphia.nodes.setup.get_large`** — `generate_personas` is the first heavyweight call in `setup.py`; without patching it a test would hit real Bedrock (CLAUDE.md convention).
- **No new dependency, no `/awos:hire`** — `langgraph-agentic` + `testing` cover every task; the reveal is text in the existing end-of-game message (no new widget, no `textual-tui`); verification is offline `uv run pytest -q`.
