# Tasks: Configurable Role Counts (Spec 014)

Vertical slices for [spec 014](./functional-spec.md) per its
[technical-considerations](./technical-considerations.md) — Phase 5 / Setup
Flexibility. Replace the fixed 7-player lineup (5 Citizens + 2 Mafiosos) with
**pre-launch-configured** counts (whole table, human included), roles dealt
**randomly**, **default 5+2 when unset**, invalid → **fail-fast exit at
startup**; the lineup is recorded in each eval run and shown in the viewer.

Each slice leaves the game runnable: Slice 1 adds config + validation (default
lineup unchanged); Slice 2 makes a *configured* lineup actually deal and play;
Slice 3 records it in the eval/viewer. Offline verification is `uv run pytest -q`
(the `safe_llm` net + the `fake_small`/`fake_large` fixtures keep the suite
model-free). The **central invariant** to hold throughout: the role deck size
(`num_mafia + num_citizens`) and the roster size (`1 human + total−1 AI names`)
both derive from one config, so they can never drift. Agents: `python-backend`
(config, setup node, Roster/prompt, harness, viewer), `testing` (suites).

## Notes for the implementer

- **The deck↔roster size invariant is the spine:** both derive from `config.num_citizens + config.num_mafia`; `_coerce_to_count` guarantees the roster is exactly `total−1`, which is what makes `len(deck) == len(players)` hold. A regression here is an `IndexError` in `assign_roles`.
- **No game-flow/graph/schema change** — `DayAction`/`Ballot` untouched; the Mafia teammate-intro and `check_win_condition` are already count-agnostic.
- **The `_MAX_TABLE_SIZE = 12` cap** is a documented ceiling (window-fit + small-model reliability + cost), trivially raised — but if raised, `_CONTEXT_WINDOW` must be re-derived from `total`, not left at 30.
- **No new dependency, no `/awos:hire`:** `python-backend` + `testing` cover every task.
